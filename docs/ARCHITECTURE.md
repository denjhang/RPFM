# RPFM 架构设计文档

> 记录时间：2026-05-12
> 硬件：RP2350A 核心板 + RESPFM Bus Board v0.7
> 目标：多芯片 FM 音源播放器下位机

## 1. 系统定位

RPFM 是 **下位机**，专门负责音频芯片驱动。不处理文件系统、不管理存储、不跑复杂 UI。

```
PC/ESP32 (上位机) ──→ RPFM (下位机) ──→ YM2413/YM2151
```

上位机负责：文件管理、UI 交互、WiFi/Bluetooth 通信
下位机负责：SPFM 总线驱动、PIO 时序、实时音频流处理

## 2. 通信协议

### 2.1 方案对比

| 方式 | 速度 | 引脚 | 说明 |
|------|------|------|------|
| USB CDC | 12Mbps | D+/D- | 当前在用，适合开发调试 |
| UART 1.5M | 1.5M baud | TX/RX = 2 | 兼容 SPFM 协议，直接复用上位机 |
| SPI | 10M+ | SCK/MOSI/MISO/CS = 4 | 适合大数据量流式传输 |

### 2.2 推荐方案

**UART 1.5M**：与 SPFM 原有协议兼容，Denjhang Music Player 上位机无需改动。VGM 数据包通过 UART 发送到 RP2350，RP2350 解码后写入音频芯片。

后续可升级到 **SPI** 以实现更高带宽（如多通道 PCM 混音实时流）。

## 3. 存储架构

### 3.1 板载 QSPI Flash（16MB）

- 非 PSRAM，不能当运行内存用
- BOOTSEL 模式下显示 128MB 是 USB MSC 协议的虚拟大小，实际物理容量 16MB
- 适合存固件代码（~100KB）+ 预置测试曲（~1MB）
- 运行时不能擦写 XIP 区域，数据分区需要代码拷贝到 RAM 再写
- 10 万次擦写寿命，不适合频繁写入

### 3.2 外挂 PSRAM（未来扩展）

| 型号 | 容量 | 接口 |
|------|------|------|
| ESP-PSRAM64H | 64MB | SPI/QPI |

PSRAM 是真正的运行内存，用于：
- YM2612 PCM 数据加载缓冲
- VGM 流式播放环形缓冲
- ILI9341 帧缓冲（320x240x2 = 150KB）
- 多通道混音中间数据

### 3.3 结论

当前阶段不需要 PSRAM，RP2350A 的 **520KB SRAM** 足够。

```
SRAM 分配（当前方案）：
  ┌─────────────────────────┐
  │ 固件代码 + 变量  ~100KB │
  │ 剩余             ~420KB │
  └─────────────────────────┘
```

### 3.4 环形缓冲区（待实现）

**问题：** 当前固件逐字节透传——收到一个 SCCI 命令就立即写寄存器。
UART/USB 速率波动直接变成时序抖动，大数据量 VGM（SNDH、多芯片）播放速度不稳定。
AY8910 已验证可播放，但大数据量 VGM 速度不稳定。

**参考：** MegaGRRL（ESP32）使用 40KB 环形缓冲区（`DRIVER_QUEUE_SIZE = 40000`）。
RP2350A 内存更紧张，需要更精简的方案。

**方案：** 下位机做环形缓冲 + 时间戳驱动播放。

```
上位机 → 写入环形缓冲区 → 下位机定时器驱动读取 → PIO 写寄存器

缓冲区大小：~16KB（参考 MegaGRRL 的 40KB 按比例缩小）
```

SRAM 分配（环形缓冲方案）：
```
┌──────────────────────────────┐
│ 固件代码 + 变量     ~100KB   │
│ VGM 环形缓冲       ~16KB    │
│ 剩余（栈、堆等）   ~404KB   │
└──────────────────────────────┘
```

缓冲区条目格式（参考 MegaGRRL MegaStream）：
```
VGM 原始字节流，下位机内嵌 VGM 命令解析器。
按 sample number (44100Hz tick) 推进时间轴。
```

流控：下位机上报缓冲区水位（空闲/半满/将满），上位机根据水位调整发送速率。
参考 MegaGRRL 的 `DRIVER_EVENT_COMMAND_HALF` 机制。

分阶段实施：
1. 先用 SCCI 逐字节透传（当前），验证所有芯片能发声
2. 加环形缓冲区 + 简单流控，解决速度不稳定
3. 改协议支持 VGM 时间戳直接传输（跳过上位机解析）

不设 SD 卡，数据全部通过 USB/UART 实时流式传输。

## 4. PIO 驱动设计

### 4.1 核心要求：连续 GPIO

PIO 指令 `out pins, N` 要求 **GPIO 编号连续**，不是物理引脚连续。

当前总线需要 14 个连续 GPIO（D0-D7 + WR# + RD# + A0-A3）：

```c
out pins, 14  // 需要 GPIO[N] 到 GPIO[N+13]
```

### 4.2 各板连续 GPIO 情况

| 板卡 | 连续 14 个空闲 GPIO | 说明 |
|------|---------------------|------|
| RP2350A 核心板 | ✅ GPIO0-13 | 14-bit 总线 + 74LVC245 电平转换 |
| RP2350 Touch LCD | ❌ 最长连续段 GPIO0-4（5 个） | GPIO 分散 |
| ESP32S3 | ❌ | 非 PIO 架构 |

### 4.3 当前接线（RP2350A 核心板）

```
PIO0 SM0 (14-bit, GPIO0-13):
GPIO0-7   → D0-D7  数据总线
GPIO8     → WR#    写信号    (1=空闲, 0=脉冲)
GPIO9     → RD#    读信号    (1=空闲, 0=读)
GPIO10    → A0     地址线0
GPIO11    → A1     地址线1
GPIO12    → A2     地址线2
GPIO13    → A3     地址线3

PIO1 SM1 (4-bit, GPIO17-20):
GPIO17    → CS0#   片选0     (1=空闲, 0=选中)
GPIO18    → CS1#   片选1
GPIO19    → CS2#   片选2
GPIO20    → CS3#   片选3

CPU 控制:
GPIO21    → IC#    复位      (上电复位一次)
GPIO25    → LED    心跳
GPIO16    → WS2812 板载 RGB  (不使用)
```

### 4.4 SPI 总线（未来共用）

ILI9341 屏幕、外挂 PSRAM、调试 UART 等外设可以共用一个 SPI 总线：

```
SPI0: ILI9341 触摸屏（预留）
  SCK      → GPIO22
  MOSI     → GPIO23
  MISO     → GPIO24
  DC       → GPIO26
  RST      → GPIO27
  CS_LCD   → GPIO28
  CS_TOUCH → GPIO29

SPI1: SD 卡（预留）
  SCK      → GPIO14
  MOSI     → GPIO15
  MISO     → GPIO30
  CS_SD    → GPIO31
```

## 5. 多芯片支持

### 5.1 SPFM 总线架构

SPFM 总线的关键在于**复用数据总线，独立片选**：

```
数据总线 (D0-D7) + WR#：所有芯片共用
A0:          YM2413 地址选择
CS0:         YM2413 片选
CS1:         YM2151 片选（未来）
A1:          YM2608 地址选择（未来）
```

PIO 程序只需写一次 WR# 脉冲逻辑，不同芯片的区别仅在 C 侧控制的 CS/A0 引脚。

### 5.2 支持计划

| 芯片 | 状态 | 说明 |
|------|------|------|
| YM2413 (OPLL) | ✅ 已驱动 | 9 通道旋律 + 5 节奏 |
| AY8910 (PSG) | ✅ 已验证 | 通过 SCCI 协议直接播放 |
| YM2151 (OPM) | 🔜 计划中 | 8 通道 FM，需注意初始化时序 |
| YM2612 (OPN2) | 🔜 计划中 | 6 通道 FM + DAC/PCM，需要 PSRAM |
| YM2203 (OPN) | 🔜 计划中 | 3 FM + 3 SSG，兼容 YM2151 类似 |
| YM2608 (OPNA) | 🔜 计划中 | 6 FM + 3 SSG + ADPCM |

## 6. 节奏通道（YM2413）

参考 libvgm 模拟器核心代码 `ym2413.c` 确认的 bit 映射：

```
寄存器 0x0E:
  bit 5 (0x20): 节奏模式启用
  bit 4 (0x10): Bass Drum  (BD)
  bit 3 (0x08): Snare Drum (SD)
  bit 2 (0x04): Tom Tom    (TOM)
  bit 1 (0x02): Top Cymbal (CYM)
  bit 0 (0x01): Hi-Hat     (HH)
```

触发节奏时只需写 0x0E 对应 bit 位即可，YM2413 内部自动从节奏乐器 ROM 加载参数。节奏音量由寄存器 0x36-0x38 控制。

## 7. 调试输出

### 7.1 USB CDC + BOOTSEL 命令

- 所有 `printf()` 输出走 USB CDC（TinyUSB 0.20.0，已替换 SDK 内置版）
- `getchar_timeout_us(0)` 轮询 CDC 输入
- 收到 `BOOTSEL\r\n` 调用 `rom_reset_usb_boot(0, 0)` 进入 BOOTSEL 模式
- 自动烧录脚本 `flash.sh`：发 BOOTSEL → 等盘符 → 复制 UF2

### 7.2 LED 指示（GPIO25）

- **数据活动**：每收到/发送一个字节，LED 亮 1ms，数据密集时连续亮
- **空闲**：无串口数据时 500ms 慢闪心跳
- 有数据时暂停心跳，只按数据实时闪烁

### 7.2 WS2812 片选指示灯（GPIO16，PIO1）

板载 4 颗 RGB LED，由 PIO1 驱动（800kHz，官方 ws2812.pio）。

| LED | 颜色 | 对应 CS |
|-----|------|---------|
| 0 | 红 | CS0 (YM2413) |
| 1 | 蓝 | CS1 (YM2151) |
| 2 | 黄 | CS2 (YM2608) |
| 3 | 绿 | CS3 (YM2612) |

选中芯片写寄存器时亮 10ms，超时自动熄灭。

### 7.3 屏幕调试信息（未来）

通过 ILI9341 显示：
- 当前播放通道和音符
- 乐器编号
- 音量电平表
- 缓冲区状态（环形缓冲占用率）
- 最后调试信息

## 8. 上位机通信架构

### 8.1 通信协议

```
上位机 → 下位机：
  [CMD 1字节] [数据 N字节]

  CMD 列表：
    0x01: 写入寄存器 (reg, data, chip)
    0x02: 开始流式传输
    0x03: 停止/复位
    0x04: 设置音乐速率（如果是跳帧）
    0x10: 请求状态（当前缓冲区占用等）

下位机 → 上位机：
  [状态 1字节] [数据 N字节]

  状态：
    0x00: 正常 / 缓冲区空闲
    0x01: 缓冲区满 / 忙
    0x10: 调试信息打印
```

### 8.2 时序要求

VGM 播放的实际时序由下位机控制。上位机发送数据包后，下位机按 YM2413 的时钟（~3.58MHz）计算等待时间。

```
上位机发送延迟 → 下位机接收 → 写入芯片 → sleep_us(等待) → 下个数据
```

下位机的 `sleep_us()` 必须精确（PIO + busy_wait_us_32），上位机只需保证数据包在缓冲区溢出前送达。

## 9. 多板协作方案

### 9.1 硬件分工

| 板子 | 角色 | 硬件（已购） |
|------|------|-------------|
| RP2350A 核心板 | 下位机：SPFM 总线驱动 | ✅ 在用 |
| ESP32-P4 触摸屏 4.3寸 | 旗舰上位机：大屏 UI + SD 卡 + WiFi | ✅ 已购 |
| ESP32-S3 触摸屏 | 便携上位机：触摸 UI + WiFi/蓝牙 | ✅ 已购 |

### 9.2 系统架构

```
ESP32-P4/S3 (上位机)                    RP2350A (下位机)
┌─────────────────────┐                ┌──────────────────┐
│ SD 卡读 VGP/VGM     │                │ PIO0: 14-bit 总线 │
│ VGM 解析 + PCM 解码 │──UART/SPI──→   │ PIO1: CS + WS2812 │
│ I2S DAC (PCM 输出)  │                │ IC# 全局复位       │
│ LVGL 触摸屏 UI      │                └────────┬─────────┘
│ WiFi/蓝牙           │                         │
│ USB MIDI            │                         ▼
└─────────────────────┘                YM2413/YM2151/YM2612...
                                              │
                                         模拟音频直出
                                              │
ESP32 I2S DAC ──────────────────────→ 模拟混合 → 功放 → 喇叭
```

### 9.3 音频同步方案

VGP 文件 = VGM（FM 寄存器写入）+ 预渲染 PCM 流。

同步方式：**上位机统一推进时间轴，FM 和 PCM 在模拟域混合。**

```
VGP 文件
  ├── FM 寄存器写入 ──→ UART/SPI → RP2350A → 硬件芯片 → 模拟音频 A
  └── PCM 采样点    ──→ I2S DAC (ESP32)        → 模拟音频 B
                                                    │
                                              A + B 运放混合 → 功放
```

- ESP32 用硬件定时器统一驱动 VGM tick（通常 44100Hz）
- FM 寄存器写入按 tick 时间戳通过 UART/SPI 发给下位机
- PCM 采样点按 tick 送 ESP32 自己的 I2S DAC
- 两路模拟信号在运放叠加，天然同步

实时软件模拟 PCM（如 YM2612 DAC 通道）方案相同，只是 PCM 来源从预渲染变成实时模拟器输出。

### 9.4 板间通信

| 阶段 | 方式 | 速率 | 引脚 | 说明 |
|------|------|------|------|------|
| 当前调试 | USB CDC | 12Mbps | D+/D- | PC 直连，RPFM 固件已支持 |
| 产品第一阶段 | UART | 1.5M baud | TX/RX = 2 | 兼容 SPFM 协议，固件不用改 |
| 未来多芯片 | SPI | 10M+ | SCK/MOSI/MISO/CS = 4 | 高带宽，适合 YM2612 DAC + YM2608 ADPCM |

UART 优先：与 SPFM 原有协议兼容，RPFM 固件无需修改，ESP32-P4/S3 的 UART 都支持 1.5M baud。
多芯片高密度数据（YM2612 DAC 实时流、YM2608 ADPCM）时再升级 SPI。

### 9.5 场景：PC + RP2350A（调试阶段，当前）

```
PC ──USB CDC──→ RP2350A ──PIO──→ YM2413
```

PC 运行 Denjhang Music Player，通过 USB CDC 发送 VGM 数据。
下位机只负责解码和时序驱动。

### 9.6 场景：ESP32-S3 + RP2350A（便携播放）

```
iPad/手机 ──WiFi──→ ESP32-S3 ──UART 1.5M──→ RP2350A ──PIO──→ YM2413
                         │
                         ├── 触摸屏 UI（LVGL）
                         ├── I2S DAC（PCM 输出）
                         └── WiFi/蓝牙接收曲库
```

ESP32-S3 做便携上位机，240MHz 双核够用。

### 9.7 场景：ESP32-P4 + RP2350A（旗舰播放器）

```
ESP32-P4 ──UART/SPI──→ RP2350A ──PIO──→ YM2151/YM2413/YM2612
   │
   ├── 4.3寸触摸屏（LVGL）
   ├── SD 卡（曲库）
   ├── I2S DAC（PCM 输出）
   ├── WiFi（无线传输）
   └── USB MIDI
```

ESP32-P4 做旗舰主控，400MHz 双核游刃有余，支持多芯片复杂 VGM。

## 10. GPIO 需求总表（RP2350A 下位机）

| 外设 | GPIO 数 | 连续性要求 |
|------|---------|-----------|
| PIO0 总线 D0-D7 + WR# + RD# + A0-A3 | 14 | 必须连续 |
| PIO1 CS0-CS3 | 4 | 必须连续 |
| IC# 复位 | 1 | CPU 控制 |
| LED 心跳 | 1 | CPU 控制 |
| WS2812 板载 | 1 | 固定 GPIO16 |
| UART TX/RX（上位机通信） | 2 | 不要求 |
| **合计** | **23** | |

RP2350A（30 GPIO: GPIO0-29）剩余 7 个，未来可接 I2S DAC 做下位机本地 PCM 输出。

## 11. 已知问题

- YM2413 频率表已验证与 libvgm 参考一致，待实际听觉验证
- 节奏通道已修正 bit 映射，待实际听觉验证
- 16MB QSPI flash 容量有限，流式播放依赖串口传输
- PIO 总线驱动目前只验证了 YM2413，未验证 YM2151 等芯片
- 74LVC245 电平转换 3.3V ↔ 5V 尚未实际接线测试
- IC# 复位需要 50ms 低电平（1ms 不够），见 docs/IC_RESET.md
- BOOTSEL 串口命令未在主循环中调用（待修复）
